# ADR History

This directory centralizes the repository's historical context, architecture decisions, execution plans, and topic discussions.

Directory conventions:

- `repository/`: repository-level architecture and decision history
- `packages/`: package-level design and evolution records
- `practices/`: designs and plans for practice examples

Organization principles:

- Use one directory per topic, preserving source materials such as `goal.md`, `adr.md`, and `plan.md`.
- Keep context files such as example sources, drafts, and acceptance records with their topic or package.
- When adding historical documents, place them under the matching subdirectory of `repository/`, `packages/`, or `practices/` instead of creating scattered `docs/` directories elsewhere in the repository.
