# ADR-001: Agent 与 CLI 解耦，Agent 只操作 Builder

状态：Accepted
日期：2026-04-26

## 背景

VBI 当前包含可开源的原子能力和面向具体应用的平台能力。

- `packages/vbi` 负责 VBIChartDSL、VBIReportDSL、Builder 与 DSL 转换。
- `packages/vquery` 负责 QueryDSL 到 SQL。
- `packages/vseed` 负责 VSeedDSL 到 VChart/VTable Spec。
- `apps/vbi_provider`、`apps/vbi_fe`、`apps/vbi_be` 负责平台资源、远端协同、鉴权与具体业务流程。
- `apps/vbi_agent` 当前同时包含 agent runtime、模型 provider、TUI、平台 provider 接入和 builder 操作工具。

这会导致 agent 能力和平台实现耦合，进而影响 `packages` 中原子组件的独立开源与复用。

## 决策

将 Agent 与 CLI 进行职责解耦。

Agent 作为独立包存在，只负责围绕 Builder 工作：

- 获取调用方注入的 Builder 工作区。
- 读取和修改 `VBIChartBuilder`、`VBIReportBuilder`。
- 生成或检查 VBIChartDSL、VQueryDSL、VSeedDSL。
- 提供 builder 级工具、上下文说明和 agent runtime 抽象。

CLI 作为应用壳存在，负责平台和业务集成：

- 对接模型 provider。
- 对接 `@visactor/vbi-provider` 或其他平台 provider。
- 处理资源选择、资源创建、资源删除、权限、鉴权和环境变量。
- 打开远端资源并将 Builder 注入 Agent。
- 承载 TUI、命令解析和具体 app 业务流程。

推荐的依赖方向：

```text
apps/vbi_agent
  -> packages/vbi-agent
  -> apps/vbi_provider 或 @visactor/vbi-provider
  -> 模型 provider SDK

packages/vbi-agent
  -> @visactor/vbi

@visactor/vbi
  -> @visactor/vquery
  -> @visactor/vseed
```

`packages/vbi-agent` 不依赖平台 provider，不直接感知 `resourceId`、HTTP、Hocuspocus、鉴权、资源列表或资源 CRUD。

## 核心接口

Agent 通过抽象工作区获取 Builder，而不是直接创建平台 client。

```ts
export interface VBIBuilderWorkspace {
  getActiveChartBuilder(): Promise<VBIChartBuilder>
  getActiveReportBuilder?(): Promise<VBIReportBuilder>
  close?(): Promise<void>
}

export interface VBIBuilderAgentInput {
  workspace: VBIBuilderWorkspace
}
```

CLI 将平台资源适配成 `VBIBuilderWorkspace`：

```ts
const provider = client.chart(resourceId)

const workspace: VBIBuilderWorkspace = {
  getActiveChartBuilder: () => provider.open(),
  close: () => provider.close(),
}
```

## 后果

正向影响：

- `packages/vbi`、`packages/vquery`、`packages/vseed` 保持平台无关，便于独立开源。
- `packages/vbi-agent` 可以作为 builder 操作层独立复用。
- CLI 可以按不同平台、模型 provider、权限系统进行组合。
- 平台资源逻辑不会污染 Builder 与 Agent 的核心能力。

代价：

- CLI 需要显式实现平台到 Builder 的适配层。
- Agent 无法直接做资源 CRUD，除非 CLI 额外注入平台工具。
- 需要迁移现有 `apps/vbi_agent` 中混合的 runtime、provider、TUI 和工具代码。

## 迁移计划

1. 在现有 `apps/vbi_agent` 内先抽出 `VBIBuilderWorkspace` 接口。
2. 将 builder 操作工具改为依赖 `VBIBuilderWorkspace` 注入。
3. 将 `runtime`、`history`、`activity-log`、agent 类型和 builder 工具迁移到 `packages/vbi-agent`。
4. 保留 `cli.ts`、`parse.ts`、`tui/*`、模型 provider 和平台 provider 接入在 `apps/vbi_agent`。
5. 为 `packages/vbi-agent` 增加不依赖平台 provider 的单元测试。
